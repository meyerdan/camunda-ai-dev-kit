# Camunda AI Dev Kit

Before doing any Camunda work, read `docs/camunda-dev-guide.md` — it contains all Camunda-specific patterns and conventions.

## Key docs
- `docs/camunda-dev-guide.md` — core Camunda 8 knowledge (BPMN, connectors, DMN, forms, CLI, API)
- `docs/env-c8run.md` / `docs/env-saas.md` — environment config
- `docs/lang-nodejs.md` / `docs/lang-java.md` — worker SDK patterns

## Skills
Use `/new-project`, `/new-process`, `/new-agent`, `/new-dmn`, `/new-form`, `/new-worker`, `/deploy`, `/start`, `/status`, `/view-process`, `/setup-environment`.

## Tools
- `c8ctl` (`c8`) — Camunda CLI for deploy, start, watch, status
- `tools/camunda-viewer/` — local file viewer (BPMN, DMN, Forms)
- Camunda Docs MCP — semantic search over Camunda docs (auto-configured by plugin)
